git hubの簡単な使い方

  • リポジトリの作り方

    • まず最初に、サイトにある緑の「New repository」というボタンを押して名前を入力する。
    Initialize this repository with a READ.MEというチェックボックスにはチェックを入れないようにしておく。「Create repository」という緑のボタンを押す。

  • git shellの準備

    • リポジトリを作った直後のページに記述されている「…or create a new repository on the command line」とある項目のechoと書かれた行より下の５行を、一行ずつgit shellにコピーアンドペーストしていけばgit shellが使えるようになる。

  • ローカルへの落とし方

    • git pullして、あとは書くだけ。

  • READMEのかきかた

    • README.mdとは、そのプロジェクトのツールの使い方や宣言などを書き、新規参加のメンバーらの道しるべとなるものである。
    基本的には、ソフト名、制作日、種類、連絡先などを記述する。丈夫にはこのような項目を箇条書きで書き、それより下に著作権及び免責事項などを書く。

  • コミットにかけての流れ

    • git statusで改変したファイルが赤文字になっているかチェックする。赤文字は改変されたという意味。
    git add [ファイル名]したあと、git commit -m "コメント内容"でコミットする。 コメントを書き忘れるとメモ帳が起動するので、そこにコメントを書く。
    あとはgit pushしてリモートにあげる。
