git hubの簡単な使い方

  • リポジトリの作り方

    • まず最初に、サイトにある緑の「New repository」というボタンを押して名前を入力する。
    Initialize this repository with a READ.MEというチェックボックスにはチェックを入れないようにしておく。「Create repository」という緑のボタンを押す。

  • git shellの準備

    • リポジトリを作った直後のページに記述されている「…or create a new repository on the command line」とある項目のechoと書かれた行より下の５行を、一行ずつgit shellにコピーアンドペーストしていけばgit shellが使えるようになる。

  • ローカルへの落とし方

    • git pullして、あとは書くだけ。

  • READMEのかきかた

    • README.mdとは、そのプロジェクトのツールの使い方や宣言などを書き、新規参加のメンバーらの道しるべとなるものである。
    基本的には、ソフト名、制作日、種類、連絡先などを記述する。丈夫にはこのような項目を箇条書きで書き、それより下に著作権及び免責事項などを書く。

  • コミットにかけての流れ

    • git statusで改変したファイルが赤文字になっているかチェックする。赤文字は改変されたという意味。
    git add [ファイル名]したあと、git commit -m "コメント内容"でコミットする。 コメントを書き忘れるとメモ帳が起動するので、そこにコメントを書く。
    あとはgit pushしてリモートにあげる。

  • ブランチとフォークについて

    • デフォルトのマスターブランチから派生して、ブランチをつくことができる。作業を分岐することで効率的かつ高速に作業を進めることができる。
    フォークとは、既存のリポジトリの複製を自分のアカウント内に作ることである。自分のアカウントに複製することで、共有されていなくても書き込んだり編集したりできる。

  • プルリクエスト

    • ローカルで改変したファイルをリモートにあげると、webページの上のほうに「Merge pull repuest」という緑のボタンがあるのでそれを押す。
    とんだページで、コメントを書きリクエストをする。マスターはそれを受け取り、Mergeボタンの上のコメントを書き押す。

  • git hub デスクトップの使い方
    • リポジトリの作り方

      • 画面の左上にある＋マークのボタンを押すと、add,crate,cloneのボタンがある。 add : パソコン内にあるリポジトリを追加する。
      create : 新しくリポジトリを作る。
      clone : リモートにあるリポジトリをダウンロードする。

    • チェンジ画面とヒストリー画面

      • チェンジ画面では、改変されてまだコミットされてないファイルの内容が色付けされてて表示されている。 ヒストリー画面では、これまでのコミットの内容やコメントを見ることができる。

    • コミットのボタンとコメント

      • チェンジ画面の下の「summary」にはコミットする要綱を書き、下の「Description」にはコメントを書く。summaryを入力すると一番下の「Commit to work」が押せるようになるので、それを押すとコミットされて改変されたファイル名が消えて、ヒストリー画面に情報が追加される。

    • パブリッシュぼたんと、シンクロボタン

      • 初めてクローンしたリポジトリをコミットした後は、リモートに挙げるために右上にある雲マークがついた「Publich」ボタンを押す。それ以降は「Sync」というボタンになる。

    • ＋ボタンの右横にあるマークはフォークのマークで、その横ではブランチを選択することができる。
