wikiについて

リポジトリに入り、Wikiというタブを開く。 「Create the first page」という緑のボタンを押す。タイトルを入力する。上のほうの、Writeタブで入力画面を表示し、Previewタブでプレビューを表示する。

wikiで使うマークダウン記法について

  • 見出し

    • [# ]を行頭につけることで見出しになる。#は6つまでつけられる。

  • 段落

    • 空の行を間に挟むことで段落になる。

  • 改行

    • 開業したいところの前に、半角のスペースを2つ入力する。

  • コードの入力方法

    • [```]で囲むか、[~~~]で囲む。

  • テキストの装飾
    • 斜体

      • *で囲む。

    • 太字

      • **で囲む。

    • 打消し線

      • ~~で囲む。

  • 箇条書き

    • [* ]か[+ ]または[- ]で囲む。

  • 数字付き箇条書き

    • [1. ]をつけたい範囲すべてにつける。すべて１でおｋ。

  • 引用

    • [> ]を行頭につける。多重引用は[>> ]で表せる。

  • リンク

    • [[表示する文字](URL)]でリンクをはれる。

  • テーブル(表)

    • |項目１ |項目２|
    |-------|------|
    |内容１ |内容２|
    で表現できる。

ブランチモデルについて

有名なブランチモデルはgit-flowとGithub Flowの２つとかなんとか

  • git-flow

    • 5つのブランチを使う。開発者が主に使うdevelopブランチ、主要な機能を実装するためのfeatureブランチ、リリースの準備を行うreleaseブランチ、リリースしたソースコードを管理するためのmasterブランチ、リリースされたソフトウェアの緊急修正をするhotfixブランチ、など多くのブランチを要するため初心者には全く分からない。

  • Github flow

    • 新しいことに取り組むときには、masterからブランチを作る。作成したブランチは定期的にGithubにpushする。リポジトリは一個だけで共有しない。git-flowに比べて単純。このサイトを作っている手順に似てる感じ。こっちのほうが好き。
